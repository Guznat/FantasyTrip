Tutorial 1 : Advanced parameters

Open scene Assets/CamShake/Scene/Tst1
It has almost the same hierarchy as Tst0, but with some different settings and more reference objects (to make you see the rotation more clearly).

Rotation
1. Set Pos Speed of WHCameraShake of "Offset1" and "Main Camera" to (0,0,0).
2. Set Rot Speed of WHCameraShake of "Offset1" to (90,0,0).
3. Set Rot Speed of WHCameraShake of "Main Camera" to (0,180,0).

4. Run the game. You will find camera shaking in random rotations when you left click on scene.
5. You also set delay of WHCameraShake in "Main Camera" to 0.05 to see the swirl effect in rotation.

Hooke factor
   Hooke is the factor to make the object shaking as if it is attached to a spring.
   The bigger the Hooke factor is, the bigger the force is used to drag the object to its original point.
   If it is unspecified (default to 0), when game starts, it will be set to a reasonable value according to Pos Speed.
   In WHCameraShake, Hooke is used to affect directional shaking and Hooke Rot is used to affect rotational shaking.
   You can adjust the Hooke and Hooke Rot when game is running. e.g. if you change the Hooke Rot of "Main Camera" from (90,0,0) to (900,0,0), you will find the camera shaking more quickly.


Damping factor
   Damping is the factor to make the object shaking with energy lose.
   The less the Damping factor is, (the bigger the lose of energy per iteration), the quicker the shaking will stop.
   In WHCameraShake, Pos Damping is used to affect directional shaking and Rot Damping is used to affect rotational shaking.
   You can adjust the Damping factors when game is running. e.g. if you change the Rot Damping of "Main Camera" from 0.9 to 0.99, you will find the camera shaking might never stop.
